# Postlane Documentation

Postlane turns your development workflow into short and long-form content. Write code, commit it, and let Postlane draft the posts.

## How it works

1. Add a repo to the desktop app
2. Postlane reads your git history and drafts posts based on what changed
3. Edit, approve, and send — or schedule via a connected provider

## What you can post to

- LinkedIn
- Bluesky
- Mastodon
- Substack Notes
- X (via connected provider)

## Where to start

- [Installation](/installation) — get the desktop app running and add your first repo
- [Quick Start](/quick-start) — write, preview, and send your first post end to end
- [Scheduling](/scheduling) — connect a provider and queue posts
- [Analytics](/analytics) — measure engagement over time
