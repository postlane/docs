I built Postlane because I kept shipping things quietly. The friction between merging a PR and writing a post about it — even a short one — was enough to skip it almost every time. Postlane sits in your IDE as a slash command: run /draft-post, and it reads your recent commits and staged changes, then returns a draft you can edit and send to LinkedIn, X, Bluesky, Mastodon, and more.

The most deliberate design choice was keeping your code off my servers entirely. I could have routed git diffs through Postlane's backend and made the setup simpler, but your repository is the most sensitive thing you're putting into this workflow. Instead, Postlane runs as a slash command inside Claude Code. When you run /draft-post, the diff goes directly to Anthropic through your existing IDE subscription — no separate API key, no Postlane server in the chain. The tradeoff is that you need Claude Code, but it means I'm genuinely unable to read your code even if I wanted to.

This is early. I've been using Postlane daily for several months, and a small number of others are in the loop, but this is its first public launch. The core flow — commit, draft, edit, schedule — works reliably. Where I'm still iterating is voice customisation: the .postlane/voice-guide.md file lets you steer the tone, but I haven't found the right defaults yet. They currently sit somewhere between "too informal" and "too press-release."

If you've tried something like this before and dropped it, I'd genuinely like to know what broke the habit — whether that was draft quality, scheduling friction, or simply not being in a regular position to post. That's more useful to me right now than feature requests.
